### N01 — 個人首頁（重設計 F01）

**設計目標**：打開就知道我今天要做什麼，5 秒找到下一步。

**新版佈局（3 區塊）**：
```
┌─────────────────────────────────────────────┐
│  🔔 公告橫幅（最新 1 則，可關閉）             │
```

## 〔待實作〕後續功能調整

### G01 — 甘特圖依賴線不重疊

**問題**：多條直角折線在同一垂直通道時會重疊，難以辨識哪條線連到哪個任務。

**設計方向**：
- 計算每條折線的中間垂直段（x 座標），偵測同一 x 區間內有多條線時自動錯開（各偏移 4-6px）
- 可採用「分配通道」策略：依 from/to row 組合排序，依序分配不同的水平偏移量
- 另一做法：hover 任務時才高亮其依賴線，其他線降低透明度

**驗收**：
- [ ] 多條依賴線存在時，折線垂直段不重疊
- [ ] 線段仍可清楚辨識起點與終點任務

### G02 — 里程碑移除手動工時，改由日常任務加總

**問題**：目前里程碑完成記錄同時有「手動填寫工時欄位」和「日常任務時數加總」，兩者並存造成混亂。

**設計方向**：
- 移除每筆完成記錄的「工時（小時）」手動輸入欄位及「儲存」按鈕
- 工時唯一來源改為：關聯到該專案任務的日常作業（`DailyTask.linked_task_id`）加總
- 保留備註（note）欄位可選填
- 統計卡「手填工時」改為「總工時」，數值來自 `daily_task_minutes` 加總
- 後端 `MilestoneLogUpdate` 可保留 note 欄位，移除 `work_minutes` 欄位（或設為 read-only）

**驗收**：
- [ ] 完成記錄卡片上不再顯示工時輸入框和儲存按鈕
- [ ] 統計卡「總工時」正確反映關聯日常任務時數加總
- [ ] 備註欄位仍可填寫與儲存
