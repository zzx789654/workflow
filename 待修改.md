# 待修改（Secure SDLC 計畫）— V3 功能規劃藍圖

> V1/V2 已完成：Kanban、多人協作、JWT 認證、WebSocket、日常作業、範本、甘特圖、月曆
> V3 本輪：PM 評估 24 個強化功能，依優先級分三批實作

---

## 目標
以 PM 視角為 WorkFlow 規劃 24 個強化功能，產出可執行的優先級藍圖。本文件為 G1（設計）階段輸出，實作從 P1 功能開始逐批推進。

## 範圍與假設
- 技術棧：同 V1/V2（FastAPI + PostgreSQL + Redis + React/Vite + TailwindCSS + Zustand）
- 部署環境：本機 Docker Compose（已運行，V2 通過 20/20 E2E）
- 每批功能實作前重新過 G1～G4 關卡
- AI 功能（F24）先以規則引擎實作，預留 Claude API 介面

## 上線標準（Exit Criteria，三維度合一）—— 每批功能適用
- 功能：批次內所有功能 CRUD 通過；核心 UX 流程驗收通過
- 資安：Critical/High = 0；新 API 全 JWT 保護；公開分享連結有 token 隔離
- 品質：E2E Smoke Test Pass；Critical/Major 缺陷 = 0

## 〔PM〕V3 功能清單與設計方向

### 🔴 P1 — 核心體驗補強（立刻實作）

#### F01 — 全域儀表板 (Dashboard KPI)
- **需求**：登入後首頁顯示個人當日待辦/延遲/完成數量卡片、本週完成趨勢折線圖、「需我處理」任務清單
- **驗收**：KPI 卡片數字正確；趨勢圖顯示 7 天資料；「需我處理」列表可點擊跳轉
- **技術**：後端 `GET /dashboard/summary`（聚合 tasks + daily_tasks）；前端 Recharts 折線圖
- **DB**：無新 table，純聚合查詢
- [ ] 後端 API 設計與實作
- [ ] 前端 KPI 卡片 + 折線圖組件
- [ ] 路由整合（取代現有 DashboardPage）

#### F02 — 任務子任務 (Sub-tasks)
- **需求**：Task 可建立子任務；子任務完成後主任務自動計算進度%；Kanban 卡片顯示子任務完成比例
- **驗收**：SubTask CRUD；父任務進度% = 子任務完成數/總數；嵌套層級限 2 層
- **技術**：`subtasks` table（parent_task_id FK tasks.id）；前端 Task 詳情頁擴充子任務列表
- **DB**：新增 `subtasks` table 或在 tasks 加 `parent_task_id` nullable FK
- [ ] DB migration 003
- [ ] 後端 SubTask CRUD API
- [ ] 前端 TaskDetailPanel 子任務區塊

#### F03 — @Mention 通知系統
- **需求**：評論輸入 @用戶名觸發通知；通知中心顯示未讀數；點通知跳到對應任務；WebSocket 推播
- **驗收**：@mention 正確解析；通知 bell icon 顯示未讀數；已讀後數字清零
- **技術**：`notifications` table（user_id, actor_id, type, ref_id, read_at）；評論 parse @；WebSocket event `notification`
- **DB**：新增 `notifications` table
- [ ] DB migration
- [ ] 後端 notifications CRUD + 評論 parse @mention
- [ ] 前端 Notification Bell + Dropdown + Toast

#### F04 — 任務搜尋與篩選
- **需求**：Kanban 頁多條件篩選（指派人/優先度/標籤/日期）；全站搜尋任務/專案/日常作業
- **驗收**：多條件即時篩選；搜尋延遲 < 300ms；搜尋結果含所屬專案名稱
- **技術**：前端 filter store；後端 `GET /search?q=&type=task|project|daily`（PostgreSQL ILIKE）
- **DB**：無新 table；需在 tasks 加 `tsvector` full-text index（選用）
- [ ] 後端 search endpoint
- [ ] 前端 FilterBar 組件（Kanban 頁頂部）
- [ ] 全站搜尋結果頁

#### F05 — 任務自訂欄位 (Custom Fields)
- **需求**：專案可自訂任務附加欄位（文字/數字/日期/下拉）；最多 10 個；可依自訂欄位排序/篩選
- **驗收**：Admin 在專案設定新增欄位；任務表單顯示自訂欄位；欄位值儲存正確
- **技術**：`project_fields` (project_id, name, field_type, options JSONB) + `task_field_values` (task_id, field_id, value TEXT)
- **DB**：新增 2 tables
- [ ] DB migration
- [ ] 後端 CRUD APIs (project_fields, task_field_values)
- [ ] 前端 ProjectSettings 頁 + 任務表單動態欄位

#### F06 — 任務依賴關係 (Dependencies)
- **需求**：設定「A 完成後才能開始 B」；被阻擋任務顯示警示；甘特圖顯示依賴連線箭頭
- **驗收**：Finish-to-Start 依賴設定；被阻擋任務在 Kanban 顯示鎖定圖示；甘特圖 SVG 箭頭
- **技術**：`task_dependencies` (from_task_id, to_task_id, dep_type)；後端 cycle detection（防循環依賴）
- **DB**：新增 `task_dependencies` table
- [ ] DB migration
- [ ] 後端依賴 CRUD + cycle detection
- [ ] 前端甘特圖 SVG 連線 + Kanban 鎖定狀態

#### F07 — 時間追蹤 (Time Tracking)
- **需求**：任務可啟動/停止計時器記錄實際工時；可手動輸入；產生工時報表
- **驗收**：計時器 start/stop；WebSocket 廣播計時狀態；工時報表可按專案/用戶篩選
- **技術**：`time_logs` (task_id, user_id, started_at, ended_at, minutes)；後端聚合報表 API
- **DB**：新增 `time_logs` table
- [ ] DB migration
- [ ] 後端 time_logs CRUD + 報表 API
- [ ] 前端計時器組件 + 工時報表頁

- **技術**：TanStack Table；Zustand view mode state
- [ ] 前端 ListView 組件
- [ ] 前端 TableView 組件（行內編輯）

#### F10 — 重複排程任務 (Recurring)
- **需求**：任務可設定重複（每天/週/月），到期自動建立下一筆
- **驗收**：設定重複規則；後端 cron 自動建立；可修改「所有後續」或「僅此次」
- **技術**：`recurrence_rules` 欄位 + APScheduler；RRULE 格式

- **技術**：MinIO 或 local volume；`task_attachments` table；multipart upload endpoint
- [ ] DB migration
- [ ] 後端 upload endpoint + MinIO/volume 整合

- **驗收**：橫條圖顯示每人任務數；工時聚合正確；> 100% 紅色
- **技術**：後端 `/workload?period=week|month` 聚合；前端橫條圖（純 CSS）
- [ ] 後端 workload API
- [ ] 前端 WorkloadPage

#### F13 — 批次操作 (Bulk Actions)
- **需求**：列表視圖多選任務，批次更改狀態/指派人/刪除；支援 Undo（5 秒）
- **驗收**：多選 checkbox；`PATCH /tasks/bulk` API；Undo Toast
- **技術**：後端 bulk update endpoint；前端 selection state + Undo setTimeout
- [ ] 後端 bulk CRUD endpoint
- [ ] 前端 CheckboxList + BulkActionBar + Undo Toast

#### F14 — Emoji 反應 (Reactions)
- **需求**：評論可加 emoji 反應；WebSocket 即時同步
- **驗收**：點 emoji 切換；顯示反應人數；WebSocket 廣播
- **技術**：`comment_reactions` (comment_id, user_id, emoji)
- [ ] DB migration
- [ ] 後端 reactions API

#### F15 — 任務 Check-in（每日進度更新）

- **技術**：`task_checkins` (task_id, user_id, content, progress, checked_at)
- [ ] DB migration
